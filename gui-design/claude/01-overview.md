## 네비게이션

### Sidebar 네비게이션

| 아이콘 | 메뉴 | 설명 | 기능 |
|--------|------|------|------|
| 🏠 | **Home** | 대시보드 | 최근 프로젝트, 통계 |
| 🔍 | **Analyze** | 분석 실행 | 프로젝트 선택, 옵션, 진행 모니터링 |
| 🔗 | **Graph** | 그래프 시각화 | Neo4j 데이터 시각화 |
| 📊 | **Results** | 결과 조회 | 클래스명세, CRUD, 시퀀스 다이어그램 |
| 📈 | **Impact** | 영향도 분석 | 테이블/메서드 영향도 |
| ⚙️ | **Config** | 설정 관리 | .env 편집, 백엔드 설정 |
| ℹ️ | **Help** | 도움말 | 사용 설명서, 단축키, 정보 |

### 탭 네비게이션

각 페이지에서 관련 기능을 탭으로 제공:

```
┌─────────────────────────────────────┐
│ 📊 Analysis Results                 │
├─────────────────────────────────────┤
│ [Overview]  [Classes]  [Methods]  [Beans]  [Endpoints]  │
├─────────────────────────────────────┤
│ Content                             │
└─────────────────────────────────────┘
```

## 페이지 목록

### 1. Dashboard (🏠 Home)

**목적**: 전체 개요 및 빠른 접근

**주요 컴포넌트**:
- 📌 최근 프로젝트 (카드 레이아웃)
- 📊 프로젝트 통계 (차트)
- ⏱️ 마지막 분석 결과 요약
- 🚀 빠른 시작 버튼

**사용자**: 모두

---

### 2. Analysis (🔍 Analyze)

**목적**: 프로젝트 분석 실행 및 모니터링

**단계**:
1. 프로젝트 선택 또는 생성
2. 분석 옵션 설정
3. 분석 시작
4. 실시간 진행 모니터링
5. 결과 확인

**사용자**: 개발자 (주로)

---

### 3. Graph Viewer (🔗 Graph)

**목적**: 코드 구조를 그래프로 시각화

**기능**:
- 클래스/메서드/필드 노드 표현
- 호출 관계 시각화
- 필터링 및 검색
- 인터랙티브 상호작용

**사용자**: 개발자 (주로)

---

### 4. Results (📊 Results)

**목적**: 분석 결과 조회

**포함 기능**:
- 클래스 명세서
- CRUD 매트릭스
- 시퀀스 다이어그램
- 다이어그램 뷰어

**사용자**: 모두

---

### 5. Impact Analysis (📈 Impact)

**목적**: 변경 영향도 분석

**기능**:
- 테이블 영향도
- 메서드 영향도
- 호출 체인 시각화
- 리스크 등급 표시

**사용자**: PM, 아키텍트

---

### 6. Configuration (⚙️ Config)

**목적**: 시스템 설정 관리

**기능**:
- .env 파일 편집
- Neo4j 연결 설정
- 분석 옵션 기본값
- 로그 레벨 설정

**사용자**: 관리자

---

### 7. Help (ℹ️ Help)

**목적**: 사용자 지원

**내용**:
- 기능 설명서
- 키보드 단축키
- FAQ
- 버전 정보

**사용자**: 모두

## 사용자 흐름

### 시나리오 1: 새 프로젝트 분석

```
Dashboard
    ↓
[+ New Project] 클릭
    ↓
프로젝트 설정 (이름, 경로)
    ↓
Analyze 페이지 이동
    ↓
분석 옵션 설정 (--clean, --use-ai 등)
    ↓
[Start Analysis] 클릭
    ↓
분석 진행 모니터링 (실시간 로그, 진행률)
    ↓
분석 완료
    ↓
Results 페이지로 자동 이동
    ↓
결과 조회 (클래스명세, 그래프, 다이어그램)
```

### 시나리오 2: 그래프 탐색

```
Results → Graph 탭
    ↓
전체 그래프 표시
    ↓
특정 클래스 검색
    ↓
클래스 선택
    ↓
호출 관계 하이라이트
    ↓
상세 정보 패널 표시
    ↓
메서드 호출 스택 추적
```

### 시나리오 3: 영향도 분석

```
테이블 선택 (DB 스키마에서)
    ↓
영향도 분석 시작
    ↓
영향받는 SQL 조회
    ↓
SQL 호출하는 메서드 조회
    ↓
메서드 호출 체인 표시
    ↓
리포트 생성 및 내보내기
```

## 설계 원칙

### 1. 사용자 중심 (User-Centric)
- 개발자: 상세 정보, 기술적 메트릭 중시
- PM: 시각적 요약, 대시보드 중시

### 2. 일관성 (Consistency)
- 동일한 색상, 타이포그래피, 레이아웃 원칙 적용
- 표준 컴포넌트 재사용

### 3. 효율성 (Efficiency)
- 자주 사용하는 기능을 쉽게 접근
- 많은 클릭 없이 원하는 정보 조회

### 4. 유연성 (Flexibility)
- 테마 변경 (밝음/어두움)
- 레이아웃 커스터마이징 가능

### 5. 접근성 (Accessibility)
- 명확한 레이블 및 아이콘
- 키보드 네비게이션 지원
- 색상만으로 정보 전달 안 함
